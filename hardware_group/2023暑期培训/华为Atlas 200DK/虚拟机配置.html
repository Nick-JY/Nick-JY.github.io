虚拟机配置SD卡制作网络配置

PC机配置： #

1.下载Ubuntu18.04.4： #

  • 使用的虚拟机软件是VMware Workstation，这里不介绍如何下载。
  • 在Ubuntu官网下载Ubuntu18.04.4对应的iso文件：
    • 相应的网站地址：Index of /releases (ubuntu.com)
      • 物联网协会实验室有对应的iso文件。

2.装载系统： #

  • 在VMware Workstation中装载Ubuntu对应的iso文件：
    • - 选择典型安装。
    • - 导入对应的iso文件。
    • - 设置用户名和密码。
    • - 选择虚拟机的安装位置，**安装位置最好不选在C盘**。
    • - 磁盘空间根据自己电脑容量选择，并且将虚拟磁盘存储为单个文件。
    • - 内存大小最好选择4GB。
    • 等待虚拟机安装......
    • 安装完成：

3.对虚拟机进行相关配置： #

  • root账户设置密码：

    • sudo passwd root
    • 设置密码。

  • 更换软件源：

    • 由于我们使用的是Ubuntu18.04，因此我们导入18.04软件源：

      • 切换管理员：

        • su root

      • 备份**/etc/apt/sources.list**文件：

        • mv /etc/apt/sources.list /etc/apt/sourses.list.bak

      • 新建sources.list文件并写入源：

        • vi /etc/apt/sources.list

        • deb http://mirrors.aliyun.com/ubuntu/ bionic main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ bionic-security main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ bionic-updates main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ bionic-proposed main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ bionic-backports main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ bionic main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ bionic-security main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ bionic-updates main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ bionic-proposed main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ bionic-backports main restricted universe multiverse

    • 更新：

      • apt update

      • apt upgrade

      • 切换到自己的根目录下：

        • su nickaljy

  • 配置vim和Samba：

    • 下载vim：

      • sudo apt install vim

    • 对vim进行配置：

      • sudo vim ~/.vimrc

      • set number

set cursorline
"Hightlight current line
inoremap [ []<Esc>i

    • 1）安装samba： #
      sudo apt install samba
      2）修改samba的配置文件： #
      sudo vim /etc/samba/smb.conf
      3）在配置文件的最后添加共享文件夹： #
      [share]
comment = share directories
path = /home/nickaljy
public = yes
writable = yes
read only = no
browseable = yes
create mask = 0644
directory mask = 0755
workgroup = WORKGROUP

      注意，第二行中的目录填写自己的用户目录。

      4）添加一个samba用户： #
      sudo smbpasswd -a nickaljy

      这里填写自己的用户名。

      5）修改共享文件权限： #
      sudo chmod -R go+rwx /home/nickaljy

      这里填写自己的用户名。

      6）重启samba服务: #
      sudo /etc/init.d/smbd restart
      7）安装网络工具： #
      sudo apt install net-tools

      查看虚拟机的ip地址：

      ifconfig

      inet 192.168.146.128

      第二行中inet后面的数字就是虚拟机ip地址。

      8）进行网络互联： #

      在Windows中win+R，输入：\\ip\share，ip指的是刚刚内串数字，完成。

  • 安装pyenv和pyenv-virtualenv以及对应版本的python：

    • 下载相关依赖：

      • sudo apt install build-essential libssl-dev zlib1g-dev \
libbz2-dev libreadline-dev libsqlite3-dev curl \
libncursesw5-dev xz-utils tk-dev libxml2-dev libxmlsec1-dev libffi-dev liblzma-dev

    • 下载git：

      • sudo apt install git

    • 安装pyenv：

      • git clone https://github.com/pyenv/pyenv.git ~/.pyenv

      • cd ~/.pyenv

      • 进入root账户。

      • src/configure

      • make -C src

      • cd ..

      • 回到本地账户：

        • su nickaljy

      • echo 'export PYENV_ROOT="$HOME/.pyenv"' >> ~/.bashrc
echo 'command -v pyenv >/dev/null || export PATH="$PYENV_ROOT/bin:$PATH"' >> ~/.bashrc
echo 'eval "$(pyenv init -)"' >> ~/.bashrc

      • echo 'export PYENV_ROOT="$HOME/.pyenv"' >> ~/.profile
echo 'command -v pyenv >/dev/null || export PATH="$PYENV_ROOT/bin:$PATH"' >> ~/.profile
echo 'eval "$(pyenv init -)"' >> ~/.profile

      • exec "$SHELL"

      • pyenv --version

    • pyenv命令解析：

      • 可以查看pyenv可以安装的python版本：

        pyenv install --list

      • 安装所需版本的python：

        pyenv install 版本号

      • pyenv版本查看：

        pyenv version

      • 显示当前拥有的python版本：

        pyenv versions
        • 显示pyenv可以管理的所有python版本，当前使用的版本前面标有*。

      • python版本全局设置(对~目录进行全局设置)：

        pyenv global 版本号
        • 将当前整个~目录的python环境设置成对应的版本。

      • python版本shell设置(使用shell命令也是对~进行全局设置)：

        pyenv shell 版本号
        • 当关闭当前shell之后，全局的python的版本又回到之前的版本。

      • python版本本地设置(进入到某一个文件夹内，以该文件夹为范围设置对应版本的python环境)

        pyenv local 版本号

        • 卸载对应版本的python：

          pyenv uninstall 版本号

  • 安装pyenv-virtualenv：

    • 使用pyenv-virtualenv同样可以创建anaconda和miniconda的虚拟环境。

      • git clone https://github.com/pyenv/pyenv-virtualenv.git $(pyenv root)/plugins/pyenv-virtualenv

      • sudo echo 'eval "$(pyenv virtualenv-init -)"' >> ~/.bashrc

      • exec "$SHELL"

      • pyenv virtualenv --version

    • pyenv-virtualenv命令解析：

      • 创建虚拟环境：

        • 指定一个版本的python，创建一个python的虚拟环境。
        pyenv virtualenv 2.7.10 名称

      • 为当前全局python版本创建一个虚拟环境。

        pyenv virtualenv 名称

      • 注意：我们每次创建虚拟环境都会生成两个条目，比如：

        • pyenv virtualenv 1234
        • 这样就会生成一个名称为1234的python虚拟环境，这个环境的版本和当前全局版本一样，但是使用pyenv versions会出现两个1234的条目：3.7.9/envs/1234 和1234，这两个条目指向的都是一个虚拟环境，短的1234是一个符号链接，存在的意义是能够让使用者快速切换虚拟环境（不需要打完整的路径）。

      • 删除虚拟环境：

        • 删除一个python的虚拟环境。

        • pyenv uninstall 名称

    • 安装python3.7.5：

      • pyenv install 3.7.5

    • 安装pip3：

      • sudo apt install python3-pip

    • 创建虚拟环境：

      • pyenv virtualenv 3.7.5 Atlas200DK

    • 将全局设置为对应的虚拟环境：

      • pyenv global Atlas200DK
