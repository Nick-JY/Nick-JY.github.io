UART串口通信 #

1.串口： #

  • 串行通信：
    • 串行通信的英文是serial communication，与之相对应的是并行通信parallel communication。
    • 很显然，目前接触到的很多通信协议都是串行通信，即数据每次传输1bit。
  • 串口：
    • 串口的英文是serial port。
    • 上面我们说了串行通信，那么很多人会认为串口就是符合串行通信的接口，这句话完全没问题。
    • 以太网、FireWire、USB等硬件接口都是满足串行通信的接口，我们当然可以叫他们串口，但实际上，串口通常表示符合 RS-232 或相关标准（如RS-485或RS-422）的硬件接口。
      • 实际上，我们不把以太网、FireWire、USB等接口叫做串口，串口指代的就是符合RS-232、RS485等标准的接口。
      • 早期PC端的COM口(9针的D型插口)使用RS-232电气标准，因此COM口就是串口。
        • 现在的电脑已经用USB取代了串口，但是我们在使用串口调试助手这一类上位机的时候，检测到的还是COM口，这个COM口是USB接口模拟出来的一个虚拟COM口。
        • COM口图片：
  • RS-232标准：
    • 它规定了电气特性、逻辑电平、信号功能、发送过程。
    • 逻辑电平：
      • 逻辑1(MARK)=-3V～-15V，逻辑0(SPACE)=+3～+15V。
    • 编码格式：
      • 最常用的编码格式是异步启停：
        • 和UART中的帧格式是相同的。
  • TTL电平：
    • TTL英文全称是：transistor transistor logic(晶体管-晶体管逻辑)
    • TTL电平：
      • TTL电平信号规定，+5V(+3.3V)等价于逻辑"1"，0 V等价于逻辑"0"。
    • 对于大多数SoC来讲，输入输出的电平都遵循TTL电平，但是它的驱动能力和抗干扰能力差，不适合作为外部的通信标准。
      • 如果将SoC和含有COM口的PC通信，那么就需要进行TTL电平到RS-232电平的转换。

2.到底什么是UART： #

  • UART的英文：Universal Asynchronous Receiver/Transmitter，即通用异步收发器。
  • UART支持串行异步全双工通信，由于是全双工通信，因此至少要有两个引脚，TXD(发送引脚)和RXD(接收引脚)。
  • 从名字中很明显的可以看出UART是一个硬件电路(电路模块)，我们平时所说的UART传输协议指的就是异步启停这种编码格式。
  • 在嵌入式领域，串口指的是UART这个硬件接口(硬件电路)：
    • 在早期，SoC和PC进行通信的时候，是通过SoC的UART模块和PC的COM端口进行通信，PC的COM口使用RS-232标准，支持的编码格式和UART相同，都是异步启停，因此大家习惯性的把UART也叫做串口。
  • SoC和PC的通信：
    • 早期，PC支持COM端口：
      • MAX232转换芯片能够将SoC的TLL电平转换成RS-232标准电平，进而实现SoC和PC的通信。
    • 现代，PC使用USB端口替换掉COM端口：
      • CH340转换芯片能够将SoC的TTL电平转换成USB电平，进而实现SoC和PC的通信。

3.UART和USART的区别： #

  • UART的全称：Universal Asynchronous Receiver/Transmitter，即通用异步收发器。
  • USART的全称：Universal Synchronous/Asynchronous Receiver/Transmitter，即通用同步异步收发传输器。

4.UART通信过程： #

  • UART通信对象：
    • SoC与SoC之间的通信可以使用UART：
    • SoC与PC之间的通信可以使用UART。
  • UART的数据帧格式：
    • 空闲：
      • 持续高电平表示空闲。
    • 启始位：
      • 向空闲状态发送1bit低电平，开始传输数据。
    • 数据位：
      • 可以是5~8bit的数据，先发低位，再发高位，一般常见的就是8bit(一个字节)。
    • 校验位：
      • 1bit的校验位。
    • 停止位：
      • 停止位是数据传输结束的标志，可以是1/1.5/2bit(通常使用1bit)的高电平。
  • UART的正确传输：
    • 1、通信双方设置相同的波特率，常用的115200和9600波特率。
    • 2、通信双方约定数据位的位数，一般设置8bit，因为C语言中的unsigned char是8bit。
    • 3、通信双方约定是否有校验位，一般情况下都不需要加校验位(干扰不大的情况下)。
    • 4、通信双方约定停止位的bit数，一般情况下使用1bit的停止位。
  • UART数据的发送和接收：
    • UART中有一个数据寄存器DR，这个数据寄存器由两个寄存器组成，一个是发送数据寄存器TDR，另一个是接收数据寄存器RDR。
    • UART能够进行串行收发是发送移位寄存器和接收移位寄存器的功劳。

5.STC89C52RC的URAT： #

  • STC89C52RC单片机内部集成一个UART，内部有两个相互独立的发送数据寄存器TDR和接收数据寄存器RDR：
    • 发送数据寄存器只能写入但不能读出；
    • 接收数据寄存器只能读出但不能写入；
  • TDR和RDR共同构成了一个数据寄存器DR，因此这两个寄存器的地址是相同的，即数据寄存器DR的地址：
    • 在STC89C52RC中该地址是0x99，并且数据寄存器DR的名称是SBUF。
  • STC89C52RC中的UART有四种工作方式，其中两种波特率可变，另外两种波特率固定。

6.相关寄存器： #

  • 总览：

  • UART控制寄存器SCON：

    • 该寄存器用来选择UART的工作方式和某些控制功能。

      • SM0位和SM1位用来确定UART的工作方式：

        • 我们这里只考虑方式1。
          • 在该模式下，每次发送8位数据，波特率可变。

      • SM2位与UART的工作方式2和方式3相关，这里不做解释。

      • REN位：

        • 该位由软件置位。
        • 当REN = 1的时候，启动UART的接收数据寄存器，开始接受数据。
        • 当REN = 0的时候，关闭UART的接收数据寄存器。

      • TB8位和RB8位与UART的工作方式2和方式3相关，这里不做解释。

      • TI位：

        • 发送中断请求标志位。
        • 在方式0中，当串行发送数据的第8位发送结束时，由内部硬件自动置位为1，此时UART向主机发送中断请求，该位必须软件复位。

      • RI位：

        • 接收中断请求标志位。
        • 在方式0中，当串行接收数据的第8位发送结束时，由内部硬件自动置位为1，此时UART向主机发送中断请求，该位必须软件复位。

      • TI和RI的注意事项：

        • TI和RI以或逻辑向主机发送中断请求，所以在主机的中断处理函数中要判断到底是由TI引起的还是RI引起的。

  • 电源控制寄存器PCON：

      • SMOD位：
        • 波特率选择位。
        • 当SMOD = 1的时候，UART的工作模式1、2、3的波特率加倍。
        • 当SMOD = 0的时候，UART各个工作模式的波特率加倍。
      • SMOD0位：
        • 帧错误检测控制位。
        • 当SMOD0 = 1的时候，SCON.7位用于FE(帧错误检测)功能。
        • 当SMOD0 = 0的时候，SCON.7位用于SM0功能。
      • 其他位暂不考虑。

  • 从机地址控制寄存器SADEN和SADDR：

    • 我们只用到单片机和PC机进行通信，不涉及多机通信，因此用不到这两个寄存器。

  • 中断允许寄存器IE：

      • EA位：
        • CPU总中断控制位。
        • 当EA = 1的时候，CPU开放所有中断请求。
        • 当EA = 0的时候，CPU关闭所有中断请求。
      • ES位：
        • UART的中断控制位。
        • 当ES = 1的时候，允许UART向CPU发送中断请求(TI和RI两个中断请求)。
        • 当ES = 0的时候，禁止UART向CPU发送中断请求(TI和RI两个中断请求)。

7.单片机中的波特率： #

  • 当使用单片机和PC机进行通信的时候，我们要确保收发双方的波特率一定，这样才能无损的收发信息。
    • 波特率：
      • 码元的传输速率，即每秒传输多少个码元。
    • 比特率：
      • 比特的传输速率，即每秒传输多少个比特。
    • 当1个码元只携带1比特信息量时，比特率 = 波特率。
    • 当1个码元携带n比特信息量时，比特率 = n*波特率。
    • 码元和比特之间的转换：
      • 传输信号中有M种码元，每一个码元携带的比特信息量是N：
        • $N = log_{2}M$
  • 在单片机中，波特率和比特率并没有区别，因为单片机中的波特率表示的就是每秒传输的bit的个数。
  • 波特率的计算：
    • 显然，我们要使用定时器对波特率进行配置，通常情况下我们使用定时器T1或定时器T2来完成这项工作。
      • 我们只是使用定时器来计算波特率，并不会用到定时器中断。
    • 由于波特率的计算公式比较复杂，推荐直接使用软件计算，对于定时器T1，也可以直接参考下表：

  • 每隔3s，单片机向PC机发送"Hello CCNU_IOT"：

    • 问题分析：
      • 使用定时器T0来计时3s，并且在定时器T0的中断服务函数中发送数据。
      • 使用定时器T1作为波特率计算器，其中定时器T1选择模式2，也就是8位自动重装载，不需要定时器T1的中断。
        • 注意，对于该型号单片机，默认选择定时器T1作为波特率计算器，如果想选择定时器2作为波特率计算器，需要对定时器T2的寄存器进行配置。
      • 我们需要使用到串口1的中断服务。
      • 串口1的中断请求优先级高于定时器T0的中断请求。
        • 如果定时器T0的优先级高于串口1的优先级，由于定时器T0的中断服务函数在发送数据之前要循环检测上一次数据的发送情况，因此程序会陷入到定时器T0的中断服务函数中。
